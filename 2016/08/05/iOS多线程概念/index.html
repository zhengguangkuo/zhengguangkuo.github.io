iOS多线程概念

多线程

基础概念

多线程的运行原理并不是多个线程同时执行，而是CPU高速切换执行的线程，通俗的说就是一边聊天一边吃水果，说和吃这两个动作都是由嘴发出来（cpu），但是并不是同时的，而是说几句话，然后吃几口水果，然后再说几句话，就这样子，貌似边说边吃，实际也是需要排队执行。

  1. 进程：一个运行中的应用程序就是一个进程，每个进程之间相互独立，运行在其专有的受保护的内存空间，另外现在我们讲的cpu单核和多核跟进程没有关系，一个运行的程序无论在单核中还是多核中都是一个进程。
  2. 线程：程序执行的最小单元，线程是进程中的一个实体，每个进程至少有一个线程，并且进程里面的所有任务都是在线程中执行。
  3. 同步：线程按顺序执行，一个完了才执行下一个。
  4. 异步：可以在当前线程开启多个线程同时进行，不按顺序执行。
  5. 队列：执行线程的队伍结构
  6. 并发队列：同时开启多个线程，执行任务，并发队列需要使用异步函数
  7. 串行队列：一个线程，顺序执行任务

实现方案

  1. NSThread — NSRunloop
  2. GCD — Block
  3. NSOperation — NSOperationQueue

优缺点
优点：

  1. 能适当提高程序的执行效率
  2. 能适当提高资源（cpu，内存）利用率
    缺点
  3. 创建线程是有开销的，iOS下的主要成本：内核数据结构大约1k，栈空间，创建时间，如果开启大量线程会降低程序的性能。
  4. 线程越多cpu在调度上的开销越大
  5. 程序设计变得复杂，如线程间通信，多线程的数据共享等。
